#### 1 Сценарий "Установка API"

UC-01: Установка API
Актор: Пользователь
Цель: Установить API
Precondition: Программа запущена
Основной поток: 
1. Пользователь зашел во вкладку "Настройки"
2. Система предоставляет окошко с вводом API
3. Пользователь вводит API
4. Система обращается к API
5. Система проверяет верный ли API
6. Система проверяет отвечает ли провайдер
7. Система проверяет есть ли на балансе свободные токены
8. Система шифрует и сохраняет API-ключ.
Альтернативные потоки: 
4А (API не верный) - Система выдает ошибку - "Неверный API-ключ" и переносит пользователя на 2 этап.
5А (Провайдер не отвечает) - Система выдает ошибку - "Провайдер не отвечает" и переносит пользователя на 2 этап.
6А (Свободных токенов нет) - Система выдает ошибку - "Кончился билинг" и переносит пользователя на 2 этап.
Postcondition: API-ключ сохранён, система готова отправлять запросы к LLM.

#### 2 Сценарий "Настройка итерации"

UC-02: Настройка итерации
Актор: Пользователь
Цель: Настроить итерацию
Precondition: Программа запущена, чат создан и открыт
Основной поток: 
1. Пользователь нажимает на "Настройки итерации"
2. Система открывает настройки итерации
3. Пользователь вводит количество итераций
4. Система принимает значения
Альтернативные потоки: 
3А (Не положительно число) - Система выдает ошибку - "Введите число от 1" и переносит пользователя в настройки итерации
Postcondition: Количество итераций задано

#### 3 Сценарий "Общение"

UC-03: Общение с LLM
Актор: Пользователь
Цель: Получить ответ от LLM
Precondition: Программа запущена, API подключен, чат создан и открыт
Основной поток: 
1. Пользователь вводит запрос в окно ввода
2. Пользователь нажимает кнопку "отправить"
3. Система отправляет запрос в АПИ
4. Система проверяет отвечает ли провайдер
5. Система проверяет есть ли на балансе свободные токены
6. Система последовательно запрашивает ответы у агентов
7. Система отображает стриминг ответа каждого агента
Альтернативные потоки: 
4А (Провайдер не отвечает) - Система выдает ошибку - "Провайдер не отвечает" и переносит пользователя на 2 этап.
5А (Свободных токенов нет) - Система выдает ошибку - "Кончился билинг" и переносит пользователя на 2 этап.
7А (Лимит токенов) - Система выдает ошибку - "Значение превышает лимит токенов"
Postcondition: Система показала ответ, итерация завершена

#### 4 Сценарий "Остановка дискуссии"

UC-04: Остановка дискуссии
Актор: Пользователь
Цель: Остановить дискуссию
Precondition: Дискуссия в процессе, итерация в статусе In_Process
Основной поток: 
1. Пользователь нажимает кнпоку остановки
2. Система выдает плашку с тремя выборами
3. Пользователь выбирает "Продолжить"
4. Система продолжает работу агентов
Альтернативные потоки: 
3А (Завершить + Синтезатор) - Итерация закрывается, Синтезатор запускается немедленно
3Б (Завершить и остановить) - Итерация закрывается, сессия уходит в паузу. Синтезатор можно вызвать позже
Postcondition: 
- Дискуссия продолжается
- 3А: Итерация Synthesized, Синтезатор запущен
- 3Б: Итерация Stopped, сессия на паузе
