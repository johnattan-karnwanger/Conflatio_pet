Система должна обеспечивать следующие функции.

##### 1. Работа с API и ошибками

1. Проверять корректность API-ключа перед выполнением запросов к LLM-провайдеру.
2. Отображать пользователю ошибки API, включая HTTP 400, HTTP 402 и HTTP 429.
3. Отображать пользователю сообщение о недоступности API-провайдера при сетевой или серверной ошибке.
4. Отображать пользователю ошибку переполнения контекста.

##### 2. Управление агентами

5. Предоставлять пользователю возможность включать и отключать агентов дискуссии.
6. Предоставлять пользователю возможность назначать модель агенту.
7. Предоставлять пользователю возможность задавать порядок ответов агентов.
8. Предоставлять пользователю возможность задавать, изменять и удалять системные промпты агентов.
9. Запускать Синтезатор после получения хотя бы одного ответа от агента дискуссии.

##### 3. Управление итерациями

10. Предоставлять пользователю возможность заранее задавать количество итераций.
11. Предоставлять пользователю возможность вручную останавливать итерации.
12. Предоставлять пользователю возможность запускать дискуссию без лимита итераций.
13. Выводить предупреждение, если пользователь останавливает систему до того, как ответили все активные агенты дискуссии.
14. Предоставлять пользователю возможность продолжить обсуждение после предупреждения.
15. Предоставлять пользователю возможность завершить обсуждение и запустить Синтезатор.
16. Предоставлять пользователю возможность завершить обсуждение без немедленного запуска Синтезатора.

##### 4. Работа с логом и контекстом

17. Сохранять историю пользовательской сессии локально в JSON-файле.
18. Автоматически выполнять саммаризацию старой части лога при достижении порогового значения токенов.
19. Передавать в API только последние N токенов из актуального состояния лога.
20. Предоставлять пользователю возможность настраивать порог саммаризации.
21. Предоставлять пользователю возможность настраивать размер скользящего окна.
22. Предоставлять пользователю возможность просматривать архив сессий.
23. Предоставлять пользователю возможность экспортировать лог сессии.
24. Предоставлять пользователю возможность очищать архив сессий.

##### 5. Настройки и отображение

25. Предоставлять пользователю возможность вводить и изменять API-ключ.
26. Предоставлять пользователю возможность видеть потоковую выдачу ответов агентов и Синтезатора.
