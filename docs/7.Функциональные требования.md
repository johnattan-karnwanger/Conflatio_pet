Система должна обеспечивать следующие функции:
##### 1. Обработка ошибок
1. Сообщать пользователю об ошибке, если введён неверный API-ключ.
2. Сообщать пользователю об ошибках API, включая HTTP 429 и HTTP 402.
3. Сообщать пользователю об ошибке, если API-провайдер не отвечает.
4. Cообщать о переполнении контекста (ошибка API HTTP 400)
##### 2. Управление агентами, моделями и ролями
4. Запускать синтезатор, если ответил хотя бы один агент.
5. Предоставлять пользователю возможность задавать порядок ответов агентов.
6. Предоставлять пользователю возможность включать и отключать любые модели.
7. Предоставлять пользователю возможность включать и отключать любые роли.
##### 3. Работа с логом и контекстом
8. Автоматически саммаризировать лог при достижении порогового значения токенов.
9. Передавать в API только последние N токенов из лога.
10. Предоставлять пользователю возможность настраивать порог саммаризации.
11. Предоставлять пользователю возможность настраивать размер скользящего окна.
##### 4. Управление итерациями и ходом дискуссии
12. Предоставлять пользователю возможность заранее задавать количество итераций.
13. Предоставлять пользователю возможность вручную останавливать итерации.
14. Предоставлять пользователю возможность запускать дискуссию без лимита итераций.
15. Выводить окно предупреждения, если пользователь останавливает систему до того, как ответили N агентов.
16. Предоставлять пользователю возможность продолжить обсуждение.
17. Предоставлять пользователю возможность завершить обсуждение и синтезировать ответ.
18. Предоставлять пользователю возможность завершить обсуждение и остановить процесс.
##### 5. Работа с файлами
19. Создавать JSON-файлы.
20. Читать JSON-файлы.
21. Записывать JSON-файлы.
##### 6. Настройка системного промпта
22. Предоставлять пользователю возможность задавать системный промпт.
23. Предоставлять пользователю возможность изменять системный промпт.
24. Предоставлять пользователю возможность удалять системный промпт.
##### 7. Работа с API-ключом
25. Предоставлять пользователю возможность вставки API-ключа.
##### 8. Отображение ответа
26. Предоставлять пользователю возможность видеть стриминг выдачи ответов.
