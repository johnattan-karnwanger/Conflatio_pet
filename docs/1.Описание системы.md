
#### Что это за продукт
Conflatio — сервис для мультиагентной системы, которая предоставляет возможность использовать несколько LLM-моделей для решения задач, требующих глубокого многостороннего анализа. Количество агентов и их роли настраиваются пользователем. Базово система поставляется с 4 агентами, но это не ограничение. Система организует структурированную дискуссию между агентами с разными ролями. 

#### Какую проблему решает
Языковые модели склонны к галлюцинациям и однобоким выводам — они не рассуждают с разных позиций и не критикуют собственные идеи. Технически то же самое можно сделать вручную в одном чате, задавая роли самостоятельно. Conflatio автоматизирует этот процесс: каждый агент занимает чёткую аналитическую позицию, а система сама управляет очерёдностью, логом и итогом дискуссии.

#### Кто пользователи
Специалист которому нужно принять решение или проработать сложную задачу: аналитик, продакт, исследователь, консультант.

#### Инфа

| Роль       | Задача                                                     | Модель            |
| ---------- | ---------------------------------------------------------- | ----------------- |
| Оптимист   | Генерирует идеи, видит возможности, мыслит без ограничений | Claude Sonnet 4.6 |
| Пессимист  | Критикует идеи, ищет риски и слабые места                  | Claude Sonnet 4.6 |
| Реалист    | Оценивает что реально работает из сказанного               | Claude Opus 4.6   |
| Синтезатор | Читает весь лог, формулирует финальный взвешенный ответ    | Claude Haiku 4.6  |

1. Оптимист генерирует идеи и возможности.
2. Пессимист критикует и выявляет риски.
3. Реалист оценивает что из сказанного реально работает.
4. Система ставится на паузу — пользователь принимает решение о дальнейших действиях.

**Управление итерациями**
- **Задать N итераций заранее** — система отрабатывает N итераций и встаёт на паузу.
- **Остановить вручную** — пользователь прерывает цикл и выбирает следующее действие.
- **Запустить без лимита** — система работает до исчерпания контекстного окна или баланса.

Если пользователь останавливает систему до того как все три агента ответили, появляется предупреждение: "Агенты не закончили рассуждение. Ответ может быть неточным."

Пользователю предлагается выбор:

| Действие                  | Описание                                                                    |
| ------------------------- | --------------------------------------------------------------------------- |
| 1. Продолжить             | Агенты продолжают отвечать                                                  |
| 2. Завершить + Синтезатор | Итерация закрывается, Синтезатор запускается немедленно                     |
| 3. Завершить и остановить | Итерация закрывается, сессия уходит в паузу. Синтезатор можно вызвать позже |

Каждый вызов агента состоит из двух частей:
- **System prompt** — описание роли и инструкции поведения агента.
- **User message** — актуальный срез лога дискуссии и исходный вопрос пользователя.

Все агенты работают с единым логом дискуссии. Для защиты от переполнения контекста применяются два последовательных механизма:

**1. Авто-саммаризация (триггер по объёму)**
Когда лог достигает порогового значения (по умолчанию 10 000 токенов), бэкенд автоматически сжимает старую часть лога и перезаписывает JSON. Порог настраивается.

**2. Скользящее окно**
	После перезаписи агенты получают только последние N токенов из уже обновлённого файла (по умолчанию 15 000 токенов). Механизмы работают последовательно: сначала саммаризация обновляет JSON, затем окно берёт срез из актуального состояния. Размер окна настраивается.

Один чат хранится как единый JSON-файл локально на пк пользователя. Данные хранящиеся в JSON:

- Запросы
- Ответы агентов
- Дата и время
- ID итерации
- ID ответа
- Количество потраченных токенов
