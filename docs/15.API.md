#### 15.1. Внешняя интеграция

Conflatio интегрируется с Claude API для получения ответов агентов дискуссии и Синтезатора.

#### 15.2. Используемый API

- **Base URL:** `https://api.anthropic.com`
- **Endpoint:** `POST /v1/messages`
- **Формат обмена:** JSON
- **Тип взаимодействия:** запрос–ответ с потоковым отображением результата в UI

#### 15.3. Обязательные заголовки

- `x-api-key` — API-ключ пользователя
- `anthropic-version` — версия API
- `content-type: application/json`

#### 15.4. Логика формирования запроса

1. Backend выбирает активного агента.
2. Backend формирует `system` с описанием роли агента.
3. Backend формирует `messages` на основе исходного вопроса и актуального среза лога дискуссии.
4. Backend отправляет запрос в Claude API.
5. Ответ отображается пользователю в UI.
6. После завершения ответа данные сохраняются в локальный JSON-файл сессии.

#### 15.5. Особенности интеграции

- Messages API является stateless: в каждый новый запрос передаётся актуальный контекст текущей дискуссии.
- Системный промпт передаётся через top-level параметр `system`.
- История сообщений передаётся через массив `messages`.

#### 15.6. Обработка ошибок

- **HTTP 400** — некорректный запрос или переполнение контекста.
- **HTTP 402** — недоступна оплата / исчерпан баланс.
- **HTTP 429** — превышен лимит запросов.
- **5xx / timeout / network error** — временная недоступность провайдера.

#### 15.7. Минимальный пример структуры запроса

```json
{
  "model": "claude-sonnet-4-6",
  "max_tokens": 1024,
  "system": "Ты агент-оптимист. Ищи сильные стороны и возможности.",
  "messages": [
    {
      "role": "user",
      "content": "Оцени идею продукта и предложи сильные стороны."
    }
  ]
}
