#### Производительность (Performance)
1. Создание JSON не должно превышать 300мс (уточнить после тестирования)
2. Запись JSON не должна превышать 300мс (уточнить после тестирования)
3. Задержка между получением сгенерированного токена (чанка) на Backend и его отрисовкой в UI не должна превышать 300мс

#### Безопасность (Security)
4. API-ключ должен быть зашифрован

#### Удобство (Usability)
5. Новый пользователь должен запустить первую дискуссию не более чем за N минут без инструкции (уточнить после тестирования)
6. Открытие существующего чата должно требовать не более 2 кликов

#### Надёжность (Reliability)
7. JSON-файл должен оставаться валидным при аварийном завершении приложения
8. Данные, записанные до сбоя API, не должны быть утеряны или повреждены

#### Поддерживаемость (Maintainability)

9. В логах должна содержаться информация о дате и времени, компоненте, причине ошибки
