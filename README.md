# Проект Conflatio: Архитектура системы многоагентных дискуссий

Conflatio — это мой пет-проект по проектированию десктопного приложения, которое помогает получать от нейросетей более глубокие и критические ответы. Основная идея в том, чтобы не просто спрашивать «что-то» у модели, а заставить несколько агентов спорить между собой, прежде чем выдать итоговый результат.

## Как это работает
Система прогоняет запрос пользователя через цепочку из четырех ролей:
1. Оптимист находит плюсы и возможности.
2. Пессимист ищет риски и критикует слабые места.
3. Реалист оценивает, насколько это вообще применимо в жизни.
4. Синтезатор собирает весь этот спор в один сбалансированный ответ.

Такой подход помогает избежать галлюцинаций нейронки и дает более объемную картинку для принятия решений.

## Что я сделал в рамках проектирования
Я прошел весь путь от идеи до детального ТЗ, готового к разработке. В репозитории лежат 17 документов, где я разложил по полочкам всё: от схем движения данных до обработки ошибок API.

### Главные задачи, которые пришлось решить:
* Защита данных: Чтобы локальный файл лога не "бился" при резком закрытии программы, я прописал логику записи только после того, как агент закончит ответ. Пока текст стримится — он живет только в памяти.
* Лимиты контекста: В первой версии (MVP) я отказался от сложной саммаризации в пользу прозрачности. Если контекст переполняется, система просто выдает понятную ошибку и предлагает пользователю самому почистить историю.
* Проверка качества: Если дискуссия прервана на полпути (например, ответил только один агент), Синтезатор предупредит, что вывод может быть однобоким. Это защищает главную ценность продукта — объективность.

## Стек (что я закладывал при проектировании)
Приложение задумывалось как десктопное (Electron + React). Для хранения данных используются обычные JSON-файлы, чтобы пользователю было легко переносить свои чаты, а для "мозгов" системы выбрано API Anthropic Claude.
